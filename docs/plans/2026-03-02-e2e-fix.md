# LexiBel E2E Fix — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Rendre LexiBel fonctionnel de bout en bout (local + production) — toutes les pages affichent des vraies données depuis l'API.

**Root causes identifiés:**
1. `NEXT_PUBLIC_API_URL` n'est pas passée comme build-arg dans CI → l'image web a `http://localhost:8000` hardcodé → tous les appels API du browser échouent en prod
2. `docker-compose.prod.yml` manque : `OAUTH_ENCRYPTION_KEY`, creds MinIO, volume SSL Let's Encrypt pour nginx
3. Pas de `.env` de prod documenté ni de script de setup → onboarding impossible
4. Bootstrap (tenant + admin) non automatisé au démarrage → DB vide si jamais lancé
5. `docker-compose.yml` dev monte les sources en volume mais sert le build de prod → pas de hot-reload

**Tech Stack:** FastAPI, Next.js 14, PostgreSQL 16 + RLS, Redis, Docker Compose, GitHub Actions

## Task 1 — Fix CI: NEXT_PUBLIC_API_URL au build time

**C'est le bug #1 : l'image web en prod a localhost:8000 au lieu du domaine prod.**

**Files:**
- Modify: `.github/workflows/ci.yml`

**Step 1: Ajouter le build-arg NEXT_PUBLIC_API_URL dans le step "Build and push Web image"**

Chercher la section `Build and push Web image` et ajouter:

```yaml
      - name: Build and push Web image
        uses: docker/build-push-action@v6
        with:
          context: .
          file: infra/docker/Dockerfile.web
          target: runner
          push: true
          tags: |
            ${{ env.WEB_IMAGE }}:latest
            ${{ env.WEB_IMAGE }}:${{ github.sha }}
          build-args: |
            NEXT_PUBLIC_API_URL=https://lexibel.clixite.cloud/api/v1
          cache-from: type=gha
          cache-to: type=gha,mode=max
```

**Step 2: Vérifier que Dockerfile.web accepte bien le build-arg**

Lire `infra/docker/Dockerfile.web` et confirmer la présence de:
```dockerfile

RUN pnpm build
```
Si absent, l'ajouter AVANT `RUN pnpm build`.

**Step 3: Commit**


**Step 2: Créer un script de génération des secrets**

Create: `scripts/gen-secrets.sh`
```bash
#!/usr/bin/env bash
# Génère un .env de prod avec des secrets sécurisés

set -e
ENV_FILE=".env.prod"

if [ -f "$ENV_FILE" ]; then
  echo "⚠️  $ENV_FILE existe déjà. Supprimer avant de re-générer."
  exit 1
fi

POSTGRES_PASSWORD=$(openssl rand -hex 32)
SECRET_KEY=$(openssl rand -hex 32)
OAUTH_ENCRYPTION_KEY=$(python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())")

cat > "$ENV_FILE" << EOF
POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
SECRET_KEY=${SECRET_KEY}
OAUTH_ENCRYPTION_KEY=${OAUTH_ENCRYPTION_KEY}
MINIO_PASSWORD=${MINIO_PASSWORD}
BOOTSTRAP_ADMIN_EMAIL=admin@lexibel.be
BOOTSTRAP_ADMIN_PASSWORD=ChangeMe$(openssl rand -hex 4)!
CORS_ORIGINS=https://lexibel.clixite.cloud
EOF

echo "✅ Secrets générés dans $ENV_FILE"
echo "   IMPORTANT: changez BOOTSTRAP_ADMIN_PASSWORD après le premier login!"
```

```python
import os, sys

def _validate_required_env() -> None:
    required = ["SECRET_KEY", "DATABASE_URL"]
    if os.getenv("ENVIRONMENT", "development") == "production":
        required += ["OAUTH_ENCRYPTION_KEY", "POSTGRES_PASSWORD"]
```

## Task 9 — Seed data : créer un dossier + contact de démo

**Pour valider E2E avec des vraies données visibles dans l'interface.**

**Files:**
- Create: `scripts/seed-demo.sh`

**Step 1: Créer le script de seed**

```bash
#!/usr/bin/env bash
# Seed data de démo pour LexiBel

set -e
API="${API_URL:-http://localhost:8000/api/v1}"

# Login
echo "→ Login..."
RESPONSE=$(curl -s -X POST "$API/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@lexibel.be","password":"LexiBel2026!"}')
TOKEN=$(echo $RESPONSE | python3 -c "import json,sys; print(json.load(sys.stdin)['access_token'])")

if [ -z "$TOKEN" ]; then
  echo "❌ Login failed: $RESPONSE"
  exit 1
fi
echo "✓ Token obtenu"

H="Authorization: Bearer $TOKEN"

# Contact de démo
echo "→ Création contact..."
curl -s -X POST "$API/contacts" -H "$H" -H "Content-Type: application/json" \
  -d '{"first_name":"Jean","last_name":"Dupont","email":"jean.dupont@example.be","phone":"+32 2 123 45 67","contact_type":"person"}' | python3 -c "import json,sys; d=json.load(sys.stdin); print(f'✓ Contact: {d[\"first_name\"]} {d[\"last_name\"]} ({d[\"id\"]})')"

# Dossier de démo
echo "→ Création dossier..."
curl -s -X POST "$API/cases" -H "$H" -H "Content-Type: application/json" \
  -d '{"reference":"2026/001","title":"Affaire de test","case_type":"conseil","status":"open","description":"Dossier de démonstration"}' | python3 -c "import json,sys; d=json.load(sys.stdin); print(f'✓ Dossier: {d[\"reference\"]} — {d[\"title\"]} ({d[\"id\"]})')"

echo ""
echo "✅ Seed terminé. Ouvrir http://localhost:3000 et vous connecter."
```
